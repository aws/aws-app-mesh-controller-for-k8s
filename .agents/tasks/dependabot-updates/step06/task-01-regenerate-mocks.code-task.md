# Task: Regenerate Mocks

## Description
Regenerate all mock files using `hack/gen_mocks.sh` to match the updated interface signatures from Steps 2–5. Verify that the generated mocks compile against the new interfaces.

## Background
The codebase has ~43 mock files generated by `mockgen`. Since interface signatures changed (EventHandler added context, admission.Decoder changed, etc.), all mocks must be regenerated. The project uses `github.com/golang/mock` — this should remain as-is for now.

## Reference Documentation
**Required:**
- Design: `.agents/planning/dependabot-updates/design/detailed-design.md`

**Note:** You MUST read the detailed design document before beginning implementation.

## Technical Requirements
1. Ensure `mockgen` is installed at a compatible version
2. Run `./hack/gen_mocks.sh` to regenerate all mocks
3. If the script doesn't cover all mock files, identify and regenerate remaining mocks manually
4. Verify all mock files compile with `make controller`
5. If `github.com/golang/mock` import path has issues, check if migration to `go.uber.org/mock` is needed

## Dependencies
- Step 5 complete (all interfaces finalized, full build compiles)

## Implementation Approach
1. Read `hack/gen_mocks.sh` to understand what it generates
2. Install/verify `mockgen` tool
3. Run `./hack/gen_mocks.sh`
4. Check for any mock files not covered by the script (look in `mocks/` and `pkg/*/mock_*.go`)
5. Run `make controller` to verify all mocks compile
6. Run `make test` to check if tests pass now

**Key locations:**
- `hack/gen_mocks.sh` (generation script)
- `mocks/` directory (generated mocks)
- `pkg/cloudmap/mock_*.go` (package-local mocks)
- `pkg/aws/services/mock_*.go` (AWS service mocks)

## Acceptance Criteria

1. **Mock Generation Succeeds**
   - Given the `hack/gen_mocks.sh` script
   - When executed
   - Then it completes without errors

2. **All Mocks Compile**
   - Given all regenerated mock files
   - When `make controller` is run
   - Then no mock-related compilation errors occur

3. **Mocks Match Current Interfaces**
   - Given the regenerated mocks
   - When reviewed
   - Then method signatures match the updated interfaces (e.g., EventHandler methods include context)
